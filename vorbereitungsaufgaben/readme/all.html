Instructions

Competitor instructions

You will be tested on your ability to develop a website running on a web server and displayed on a web browser to a minimum XHTML standard. You are expected to have a professional attitude and general good disposition.

It is suggested that you read carefully through this assignment and look through the provided content for the website such as the images and text.

Schedule

For the whole competition you will be provided with approximately seven hours. The work is grouped into three parts. In the first part you will extend a given template by programming and integrating dynamic components. In the second part you will design a logo and a prototype of a WebApp. In the third part you will template a given design PSD.

Introduction 08:15 to 08:30
Morning 08:30 to 12:00 Part one and two
Lunch break 12:00 to 13:15
Afternoon 13:15 to 17:00 Part one and two
Part three 2h (random) Templating skills

Rules & rating

Resources: Do not bring in any resources or use the Internet. The competition has to be completed with the resources provided.

Skills and material: You are free to produce your website with any skills that you possess and use any of the available tools. In addition to the material provided it is of course up to you, if you want to produce new material or change the existing one where you feel it is necessary. Working time: Probably there will not be enough time to complete all formal and technical criteria. Keep in mind that the goal of the competition is to deliver a running website.

Work order: We suggest to start with part one and continue with part two in the afternoon. However, you are free to start with tasks from part two if you wish.

The special part three will start at a random time and last for two hours.

Task 2 - Towers of Hanoi (OOP)

The customer wants to put a game on his website. Therefore he decided to implement the game Towers of Hanoi. His designer already implemented the frontend and parts of the backend.

Optionally he asked you to implement the interfaces he found on the internet. You can decide yourself, if you want to implement the game using OOP or not. For implementing it using OOP you will be rewared with bonus points.

Game rules

  • One player tries to rebuild a brick tower on one of the provided stacks of a game board.
  • The game board consists of a base with min 3 stacks and a set of min 3 bricks in decreasing size.
  • The game board can only be in square format - number of stacks == number of bricks per stack.
  • The difficulty is equals the number of stacks/bricks.
  • A brick must not be placed on a smaller one.
  • Only one brick may be moved to one of the other stacks in one go.
  • In preparation to the game place the bricks on top of each other in decreasing size on one of those stacks.
  • The idea is to use as little moves as possible to rebuild the tower on a different stack.

Working space

  • You may add additional PHP files, but only inside the tasks directory: work/part_one/tasks/*. Only files within this directory will be corrected.

Coding guideline

  • Format the code according to work/part_one/tasks/task2-coding-guideline.php in every task2 file

Hints

  • use $request->getMethod() to get the HTTP method name ('GET', 'POST')
  • use $request->query->get('test') instead of $_GET['test']
  • use $request->request->get('test') instead of $_POST['test']
  • use $session->set('varName') to set a session variable (string, array, ...)
  • use $session->get('varName') to get a session variable (string, array, ...)
  • Start new game: The user can submit the game configuration page and a new game begins - using the form on the Towers of Hanoi configuration page.

    Save the configuration and the game configuration (board) into the session. The form posts the data to the game page (method POST). Save the board into the $params array. The expected result is that the game is displayed correctly (difficulty 3 or 4).

    Use the prepared file work/part_one/tasks/task2-towers-of-hanoi.php.

    GET request parameters:
    parameter name values description
    name [string] The user name.
    difficulty 3 or 4 The difficulty of the game.
  • Gameplay:
    The frontend functionality is already implemented. You have to handle the GET calls in the backend - e.g. ?fromStackId=0&fromBrickId=1&direction=right&steps=2

    • Top bricks can be moved left and right. If an user tries to manipulate the game, a message is displayed - e.g. move a brick which is not on the top.
    • A brick can only be placed on a bigger brick. If a user tries to manipulate the game, a message is displayed - e.g. move a brick on a smaller brick.
    • When all bricks are on the rightmost stack, the user is redirected to the start page and a message is displayed. Use the session to determine if the user won.

    Program the functionality in the work/part_one/tasks/task2-towers-of-hanoi.php and work/part_one/tasks/task2-towers-of-hanoi-start.php files.

    GET request parameters:
    parameter name values description
    fromStackId 0-3 ID of the stack from left to right, starting from 0.
    fromBrickId 0-3 ID of the brick from top to bottom, starting from 0.
    direction left or right The direction in which the brick should be moved.
    steps 1-3 The amount of steps the brick should be moved.
    Attention: The brick ID given from the frontend starts from the top, where as in the stack the brick IDs start from the bottom!
  • bonus OOP: Use the classes named Board / Brick / Stack which implement the given Interfaces. You do not have to include/require the files, this is already prepared for you.

    Use the prepared class files work/part_one/tasks/Game/src/SM2014/TOH/Entity/*.php.
  • bonus Coding guidelines: The customer wants to share the game with the open source community. Therefore he asked you to follow the given coding guidelines.

  • Expected results

    Users can set up, start and play the game Towers of Hanoi.

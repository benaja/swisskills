Task 2 - Sliding Puzzle

You have to implement a 4×4 sliding puzzle game. One of those from your childhood where you had to slide the tiles to correct the order by moving around the tiles. The needed HTML and CSS files are prepared. You have to implement the functionality using JavaScript only.


Sliding game HTML

  • Working directory

    The working directory is located in work/part_two/task2/.

    You may only change the file slide.js. Add all the JavaScript functionality within this file.

  • JavaScript libraries

    You are allowed to use any of the provided JavaScript libraries placed under work/part_two/task2/libraries. However, if you write the game in plain JavaScript, you will get additional points. To use a library, you may include it in game.html on line 48.

For reviewing you work Google Chrome is used. So be sure that you are developing on Google Chrome.
Rules:
  • Only code written in the slide.js file will be rated.
  • Only modify the slide.js according to the comments in the file.
  • You are allowed to include a JavaScript library in game.html on line 48. However, you wont get the extra points if you do so.

  • Testing the game

    To support testing the functionality, the order of the initial tiles is in an easy solvable position. To solve, you can move 6, 10, 11 and 15. The first game always starts with this positioning.

  • Bonus: Save tile state in session

    You get additional points if the current tile order is persisted in the current session (a browser refresh does not change the current state). If you implement this functionality, also provide a functionality to empty the saved state from the session. Use the reset button (#resetwrapper) and unhide it using JavaScript.

  • Expected result

    A working sliding puzzle game for Google Chrome.

Marking scheme

Primary 48

  • 2 When opening game.html (new session), a new game is started and the order of the tiles corresponds to the one given in the HTML file.
  • 10 When pressing the "new game" button, the tiles are ordered randomly (4, without library 10).
  • 20 When clicking a tile, it swaps the position with the free space (4, without library 8). When clicking on a tile that cannot be moved, nothing happens (8, without libary 12).
  • 8 When the tiles are in the correct order, the "you won" message appears by unhiding #game-message (1, without library 2) and the tiles cannot be moved anymore (3, without library 6).
  • 8 After a won game, when clicking the "new game" button a new game starts: the "you won" message disappears (1, without library 2), the tiles are ordered randomly (2, without library 4) and a new game is playable (1, without library 2).

Bonus 10

  • 6 The current state is persisted over a browser refresh (3, without library 6).
  • 4 The initial order can be restored easily. Therefore the "reset" button (#resetwrapper) is unhided and working (2, without library 4).

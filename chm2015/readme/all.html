Competitor instructions

Task 2 - Sliding Puzzle

You have to implement a 4×4 sliding puzzle game. One of those from your childhood where you had to slide the tiles to correct the order by moving around the tiles. The needed HTML and CSS files are prepared. You have to implement the functionality using JavaScript only.


Sliding game HTML

  • Working directory

    The working directory is located in work/part_two/task2/.

    You may only change the file slide.js. Add all the JavaScript functionality within this file.

  • JavaScript libraries

    You are allowed to use any of the provided JavaScript libraries placed under work/part_two/task2/libraries. However, if you write the game in plain JavaScript, you will get additional points. To use a library, you may include it in game.html on line 48.

For reviewing you work Google Chrome is used. So be sure that you are developing on Google Chrome.
Rules:
  • Only code written in the slide.js file will be rated.
  • Only modify the slide.js according to the comments in the file.
  • You are allowed to include a JavaScript library in game.html on line 48. However, you wont get the extra points if you do so.

  • Testing the game

    To support testing the functionality, the order of the initial tiles is in an easy solvable position. To solve, you can move 6, 10, 11 and 15. The first game always starts with this positioning.

  • Bonus: Save tile state in session

    You get additional points if the current tile order is persisted in the current session (a browser refresh does not change the current state). If you implement this functionality, also provide a functionality to empty the saved state from the session. Use the reset button (#resetwrapper) and unhide it using JavaScript.

Task 1 - Mobile Calendar View

Create a mobile design for a calendar day view using your favorite tool (e.g. Adobe Photoshop). The design has to meet professional visual expectations.

  • Screen dimensions

    You have to design for the Samsung Galaxy S5 screen only:

    • 1920x1080px
    • Portrait format only!

    Use the Photoshop template file (template.psd) provided.

  • Color scheme

    For your design you have to choose four to six colors. Define your color scheme and place the colors used in the following color scheme file:

    color-scheme.html

    You have to modify the CSS in the color-scheme.html file according to the comments.

The following data has to be displayed:

  • General information:

    • Day number: 23
    • Day name: Wednesday
    • Month and year: September 2015

  • Action buttons: Create the following icon buttons:

    • A button to create a new calendar entry
    • A button to show the whole month
    • Buttons to change day (back and forth)

    You are expected to design necessary icons by yourself and are not allowed to use existing icons (e.g. fontawesome).

  • Calendar entries: 3 to 5 calendar entries have to be visible within the phones viewport (without scrolling). Each calendar entry has to contain a title, a time, a location and the attendees pictures. You can choose the items from the following list:

    Title Time Location Attendees
    Video Conference 8:15 AM Zurich Julia, Pascal, Dana, Jasmin
    Daily Standup 10:30 AM Zurich Manuel, Julian
    Mark's Birthday Meal 11:30 AM Lilys Restaurant Mark, Michael, Laura, Noora, Hans, +5
    Gala Night 4:00 PM Olten Emma, Kristiana
    Gala Apero 8:00 PM Olten Emma, Kristiana
    Train Home 11:00 PM Olten Julia

    Use pictures for attendees not their names. You can choose avatars from these pictures: faces1, faces2, faces3

    Create and use icons for the time and location.

The customer asked to get the RAW design file and also a PNG version which he can print and show his stakeholders.
